Read the progress directory at `.claude/docs/progress/` to understand the current state of work.

1. Read `.claude/docs/progress/current-sprint.md` for the task list
2. Read the most recent report in `.claude/docs/progress/reports/` (sort by filename, latest first)
3. Summarize:
   - What was done last session
   - What tasks remain
   - Suggest what to work on next, prioritized by importance

If no progress files exist yet, ask the user what they'd like to work on and create the initial `current-sprint.md`.

Keep the summary concise. End with a clear next action the user can approve or redirect.
